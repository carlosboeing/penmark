Checkout Service — High Level Design

This document describes the proposed architecture for the new checkout service, covering the order pipeline, payment integration and the consistency model for reads and writes.

1. Consistency model

The checkout service uses eventual consistency for the read path. Writes go through the primary store and are replicated to read replicas within an unbounded window. Clients retry on staleness.
